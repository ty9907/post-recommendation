# 文章抄袭检测功能 Spec

## Why
当前系统缺乏有效的文章抄袭检测机制，导致可能存在内容重复或高度相似的文章被发布。需要实现一个全面的两阶段抄袭检测系统，通过标签分析和内容相似度比较，准确识别潜在的抄袭案例，保护原创内容质量。

**项目范围限定**：本项目仅关注文章标签提取和文章对比相关的核心功能，不包含文章获取/采集相关的代码实现。

## What Changes
- 实现标签提取与分析模块，识别文章核心标签并进行比较
- 开发自适应标签相似度比较系统，根据文章长度动态设置阈值
- 实现多种内容相似度检测算法（TF-IDF、余弦相似度、编辑距离、SimHash等）
- 提供算法选择机制，允许用户根据需求选择合适的相似度算法
- 开发相似度评分系统，量化文章之间的相似度百分比
- 添加可配置的灵敏度设置，调整检测严格程度
- 实现检测过程和结果的清晰记录机制
- 开发误报减少机制，最小化错误标记
- 优化性能以支持大型文章数据集的高效处理

**不在范围内**：
- 文章获取/采集功能
- 文章存储的数据库实现
- 文章发布流程集成
- 外部系统集成接口

## Impact
- Affected specs: 标签提取系统
- Affected code: 
  - 新增 `com.example.demo.duplicate` 包及子模块
  - 扩展 `TagService` 以支持标签比较
  - 新增相似度计算算法实现
  - 新增检测报告生成模块
  - ArticleRepository 仅定义接口，具体实现由外部系统提供

## ADDED Requirements

### Requirement: 标签提取与分析系统
系统应提供稳健的标签提取机制，识别并提取每篇文章中的核心标签。

#### Scenario: 标签提取成功
- **WHEN** 系统接收到一篇文章内容
- **THEN** 系统应提取出文章的核心标签列表，包含标签名称、权重和频率

#### Scenario: 标签比较
- **WHEN** 系统比较两篇文章的标签
- **THEN** 系统应识别共享标签并计算标签相似度

### Requirement: 自适应标签相似度阈值
系统应根据文章内容长度动态设置共享标签阈值，用于触发进一步相似度分析。

#### Scenario: 动态阈值计算
- **WHEN** 系统需要判断是否触发内容相似度检测
- **THEN** 系统应根据文章长度动态计算最小共享标签数量阈值（短文章阈值较低，长文章阈值较高）

#### Scenario: 阈值触发
- **WHEN** 两篇文章的共享标签数量达到或超过动态计算的阈值
- **THEN** 系统应触发内容相似度检测

#### Scenario: 阈值算法
- **WHEN** 计算动态阈值时
- **THEN** 系统应考虑文章长度、标签总数、标签密度等因素，使用自适应算法计算合理的阈值

### Requirement: 多算法内容相似度检测
系统应实现多种内容相似度检测算法，允许用户根据需求选择合适的算法。

#### Scenario: 算法选择
- **WHEN** 用户配置相似度检测算法
- **THEN** 系统应提供多种算法选项（如 TF-IDF、余弦相似度、编辑距离、SimHash、Word2Vec等）

#### Scenario: 算法切换
- **WHEN** 用户切换不同的相似度算法
- **THEN** 系统应使用新选择的算法进行相似度计算

#### Scenario: 文本结构分析
- **WHEN** 系统分析文章内容
- **THEN** 系统应识别文本的段落结构、句子结构和关键特征

#### Scenario: 语义相似度计算
- **WHEN** 系统比较两篇文章的内容
- **THEN** 系统应计算语义相似度，考虑词汇、句式和上下文

### Requirement: 相似度评分系统
系统应提供评分系统，量化比较文章之间的相似度百分比。

#### Scenario: 相似度计算
- **WHEN** 系统完成两篇文章的比较
- **THEN** 系统应返回0-100%的相似度评分

#### Scenario: 多维度评分
- **WHEN** 系统计算相似度
- **THEN** 系统应提供标签相似度、文本相似度和综合相似度评分

### Requirement: 可配置灵敏度设置
系统应提供可配置的灵敏度设置，调整检测严格程度。

#### Scenario: 灵敏度配置
- **WHEN** 管理员调整检测灵敏度参数
- **THEN** 系统应根据灵敏度调整相似度阈值和判定标准

#### Scenario: 灵敏度应用
- **WHEN** 系统执行抄袭检测
- **THEN** 系统应使用当前配置的灵敏度设置进行判断

### Requirement: 检测结果记录
系统应清晰记录检测过程和结果。

#### Scenario: 检测日志
- **WHEN** 系统执行抄袭检测
- **THEN** 系统应记录检测时间、检测对象、相似度分数和判定结果

#### Scenario: 检测报告
- **WHEN** 检测完成
- **THEN** 系统应生成包含相似度分数、共享标签信息和潜在抄袭标记的清晰报告

### Requirement: 高效处理大型数据集
系统应支持高效处理大型文章数据集。

#### Scenario: 批量检测
- **WHEN** 系统执行批量抄袭检测
- **THEN** 系统应在合理时间内完成检测（100篇文章检测时间 < 10秒）

#### Scenario: 性能优化
- **WHEN** 系统处理大量文章
- **THEN** 系统应使用缓存、并行计算等优化策略提升性能

### Requirement: 误报减少机制
系统应实现误报减少机制，最小化错误的抄袭标记。

#### Scenario: 多重验证
- **WHEN** 系统检测到高相似度文章
- **THEN** 系统应使用多种算法交叉验证，降低误报率

#### Scenario: 算法组合验证
- **WHEN** 用户启用多算法验证模式
- **THEN** 系统应使用多种相似度算法同时计算，综合判断是否为抄袭

#### Scenario: 白名单机制
- **WHEN** 管理员将特定文章对加入白名单
- **THEN** 系统不应将这些文章对标记为抄袭

## MODIFIED Requirements

### Requirement: 标签提取服务扩展
现有标签提取服务需要扩展以支持标签比较功能。

#### Scenario: 标签比较接口
- **WHEN** 系统需要比较两篇文章的标签
- **THEN** TagService应提供标签比较方法，返回共享标签列表和相似度

## REMOVED Requirements
无移除的需求。
