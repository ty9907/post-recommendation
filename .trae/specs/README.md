# 规范驱动开发指南

## 概述

本项目采用规范驱动开发模式，所有重大变更都需要先创建规范文档，经过审批后再实施。

## 规范文件结构

```
.trae/
└── specs/
    └── <change-id>/
        ├── spec.md       # 规范文档：描述变更内容、原因和影响
        ├── tasks.md      # 任务列表：具体的实施步骤
        └── checklist.md  # 检查清单：验证实施结果
```

## 使用流程

### 1. 创建新规范

使用 `/spec` 命令创建新的规范：

```
/spec <功能描述>
```

例如：
- `/spec 添加用户认证功能`
- `/spec 优化标签提取性能`
- `/spec 重构数据库访问层`

### 2. 规范审批

创建规范后，系统会生成三个文档：
- **spec.md**: 详细描述变更内容
- **tasks.md**: 分解的实施任务
- **checklist.md**: 验证检查点

您需要审阅这些文档，确认后系统才会开始实施。

### 3. 实施变更

规范审批后，系统会：
1. 按照任务列表逐步实施
2. 更新任务完成状态
3. 执行检查清单验证

### 4. 完成验证

所有任务完成后，系统会：
- 执行所有检查点验证
- 确保代码质量
- 完成变更交付

## 当前项目信息

**项目名称**: post-recommendation  
**主要功能**: 标签提取系统  
**技术栈**: Java, HanLP, IKAnalyzer

### 核心模块

- **tag/**: 标签提取核心模块
  - `TagExtractor`: 标签提取器接口
  - `HanLPTagExtractor`: HanLP实现
  - `IKAnalyzerTagExtractor`: IK分词器实现
  - `SimpleTagExtractor`: 简单实现
  - `TagService`: 标签服务
  - `TagExtractionConfig`: 配置管理

## 开始使用

现在您可以创建第一个规范了！使用以下命令：

```
/spec <您的功能需求>
```

系统会引导您完成整个规范驱动开发流程。
